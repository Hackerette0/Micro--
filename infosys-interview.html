Warm-Up — High Repeat Probability ▾

Simple questions interviewers use to break the ice or test basics before the main problem • >40% chance of appearing verbally or on whiteboard • click to collapse

Must Know
Palindrome Check
Easy✓
Strings · Two Pointers
Check if a string (or number) reads the same forwards and backwards. One of the most common icebreaker problems in Indian company interviews — know both the string reverse and two-pointer approach.
LeetCode #125 GFG
Reverse a Linked List
Easy✓
Linked List · Pointers
Reverse a singly linked list iteratively and recursively. Asked in nearly every Infosys SP technical round as a baseline check — know both approaches and be ready to trace through with a diagram.
LeetCode #206 GFG
Find Middle of Linked List
Easy✓
Linked List · Slow-Fast Pointers
Find the middle node of a linked list in one pass using the slow and fast pointer technique. Interviewers specifically ask this to test whether you know Floyd's two-pointer pattern beyond just cycle detection.
LeetCode #876 GFG
First Non-Repeating Character
Easy✓
Strings · HashMap
Find the first character in a string that does not repeat. Tests frequency counting with a HashMap — a clean O(n) problem that interviewers use to check if you default to brute force or think optimally.
LeetCode #387 GFG
Missing Number (1 to N)
Easy✓
Arrays · Math
Given an array of n-1 numbers from 1 to n, find the missing one. Know the sum formula approach O(1) space — interviewers follow up asking you to avoid XOR and then asking for XOR anyway. Know both.
LeetCode #268 GFG
Armstrong Number
Easy✓
Math · Number Theory
Check if a number equals the sum of its own digits each raised to the power of the number of digits (e.g. 153 = 1³ + 5³ + 3³). A classic Infosys favourite — comes up repeatedly in screening and warm-up rounds.
GFG — Armstrong LeetCode #728 (similar)
Fibonacci (Iterative)
Easy✓
Math · DP Basics
Print or return the nth Fibonacci number. Know iterative O(1) space — if you write recursion, interviewers will immediately ask you to optimise it. Knowing memoisation on top is a bonus that impresses.
LeetCode #509 GFG
Anagram Check
Easy✓
Strings · Sorting / HashMap
Check if two strings are anagrams of each other. Know both the sort-and-compare approach and the frequency array O(n) approach — interviewers ask which is better and why. Always say frequency array.
